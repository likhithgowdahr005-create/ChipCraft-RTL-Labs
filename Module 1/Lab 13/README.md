# Lab 13 тАУ Clock Gating using D Flip-Flops

## Aim

To implement and verify **Clock Gating** using Verilog HDL by comparing the behavior of a standard D Flip-Flop and a clock-gated D Flip-Flop, and to observe the reduction in unnecessary switching activity using Verilator and GTKWave.

---

## Theory

Clock gating is a widely used low-power design technique in VLSI systems. Instead of continuously supplying the clock to every sequential element, the clock is selectively enabled only when required. Since dynamic power consumption is directly proportional to switching activity, disabling the clock during idle periods significantly reduces power consumption.

The gated clock is generated using the following Boolean expression:

```text
gated_clk = clk & enable
```

Where:

- **clk** тАУ Main system clock
- **enable** тАУ Clock gating control signal
- **gated_clk** тАУ Clock supplied to the gated flip-flop

When **enable = 0**, the gated clock remains inactive and the flip-flop retains its previous value. When **enable = 1**, the gated clock follows the main clock and normal flip-flop operation resumes.

## RTL Design

The design consists of two separate D Flip-Flops:

- **ff_no_gating.v**
  - Standard positive-edge triggered D Flip-Flop.
  - Receives the system clock directly.
  - Updates its output on every rising edge of the clock.

- **ff_with_gating.v**
  - Uses a gated clock generated by ANDing the clock with the enable signal.
  - Updates its output only when the enable signal is active.
  - Demonstrates reduced switching activity and dynamic power consumption.

## Testbench

The testbench performs the following operations:

- Generates a clock with a 10 ns period.
- Applies reset initially.
- Changes the input data.
- Enables and disables clock gating.
- Dumps waveform data into `dump.vcd`.
- Verifies both gated and non-gated flip-flop outputs.

### Waveform Observation

The waveform demonstrates the difference between a normal D Flip-Flop and a clock-gated D Flip-Flop.

- The system clock toggles continuously throughout the simulation.
- The **enable** signal controls whether the gated clock reaches the gated flip-flop.
- The **q_nogate** output updates on every positive clock edge whenever the input changes.
- The **q_gate** output changes only while the **enable** signal is high.
- When **enable** becomes low, the gated clock stops, preventing unnecessary switching and keeping the output unchanged.

This behavior clearly illustrates how clock gating minimizes dynamic switching activity while preserving the stored data.
